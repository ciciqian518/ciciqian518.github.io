⏱️ 30 秒电梯陈述

给正在做试管婴儿（IVF）的人，一个把全程用药、指标、结果管起来的「病程管家」，从最高频、最刚需的「用药提醒」切入。试管用药种类多、途径杂、时段乱、还跨多个周期，确实难记，现在只能用通用提醒 App 凑合——我们懂 IVF，把「一堆乱药」变成「一条清晰的今日时间轴」，还能让伴侣一起参与。每年 85 万新患者、医保刚全国覆盖、需求正在爆发，是个口碑驱动的中量级垂直市场。数据全存用户手机本地——隐私既是卖点、也是合规最优解。创始人本人就是真实 IVF 患者。

🧭 我怎么看这件事（一页纸全局）

做什么IVF 病程管家——把试管全程的用药 / 指标 / 结果管起来，从用药提醒切入。
切入点用药提醒：最高频、最刚需、最痛、也最轻最安全（确定性 + 免责）。
解决什么用药多途径多时段、跨周期、复杂难记；更深一层——全程缺「掌控感」、伴侣难参与。
凭什么选我① 懂 IVF 的深度（通用 App 给不了）② 创始人是真实患者（信任不可复制）③ 数据本地，隐私即卖点。
怎么挣钱免费验证 → 便宜年费解锁高级 → B2B2C 诊所推荐；不卖数据、不做引流。
怎么迭代纵向沿旅程：v1 用药 → v2 早孕 / 保胎 → v3 孕期；横切随时加。

  • 「没人做是因为不挣钱」恰是单打独斗者的机会

1 产品是什么 & 目标客群

IVF 病程管家——第一个 MVP = 用药提醒（钩子）。

不一上来做大产品，先长「用药提醒」这一个器官：把试管全程散落、多来源（不同医院 / 时间）的用药、指标、每次结果结构化记下来 + 按时间轴追踪 + 该提醒时提醒。用药带日活，数据带长期价值。

目标客群（按阶段）

  • 种子用户：复杂情况者（多次促排 / 免疫 / 三代）——最痛、可直接触达（创始人病友圈）
  • 放大期：常规夫妻（通用流程，占大盘）
  • 未来延伸：早孕保胎 → 孕期 → 母婴 / 往前到备孕前段

用户画像

  • 以女性为主，25–40 岁；焦虑、信息过载
  • 极度需要保密（职场 / 家庭 / 社会三重压力）
  • 多数有伴侣，但伴侣常「插不上手」
  • 单用户高强度活跃窗口 8–10 个月

2 价值：功能性 + 背后的情感性

功能是入口，情感才是留存和口碑的根。国外产品反馈印证：「掌控感」是这个旅程最稀缺的东西。

🛠 功能性价值

  • Aha：第一次把「一堆乱药」变成「一条清晰的今日时间轴」
  • 多药 / 多途径 / 多时段不漏不错
  • 全程数据沉淀，换医院 / 换阶段不丢失

❤️ 情感性价值（更深，决定留存与口碑）

两大支柱：

  • ① 自主掌控：第一次对自己的治疗有掌控，而不是被一堆药、报告、「下一步」推着走——这半年的掌控感。
  • ② engage 伴侣：虽有人独自承受，但「让伴侣参与进来」是绝大多数人的真实处境——让他看见你吃这么多药的不易、能搭把手。这本身就是情感价值，不是次要功能。
价值靠什么落地？→ 靠下面的具体功能 + 版本迭代（第 3、4 节）。每个功能存在的理由，都是它比竞品 / 老办法更好地兑现了上面某一层价值。

4 版本主线：功能 × 痛点 × 凭什么用我

这是 pitch 主体——每个版本讲清：核心功能 → 解决什么痛点、有多痛 → 用户现在怎么办 → 凭什么用我（而不是竞品 / 老办法）。

v1 用药提醒 · 当前 MVP（单周期） 细节最清晰

核心功能：多药 / 多途径 / 多时段排程 · IVF 专用药品库（雪诺酮 / 补佳乐 / 达必佳…含途径/剂量/储存）· 方案随治疗阶段变 · 轻量伴侣参与（分享给伴侣 / 伴侣提醒）。

痛点 & 有多痛：促排每天注射 10–14 天、移植后多种药混着（口服 / 阴道 / 注射）、高频、复杂、确实难记；漏服影响依从性，而依从性关系结果。一个周期 2–4 个月、自付 0.4–3 万，不想因为漏药添乱。

用户现在怎么办：通用提醒 App（药准时 / MedTimer）不懂 IVF · Excel / 闹钟 / 纸笔 · 群里问姐妹 · 靠记忆。创始人自己在用「药准时」凑合——用户在用劣质方案硬扛 = 最强需求信号。

凭什么用我：通用 App 排不出 IVF 的多途径多时段、不懂治疗阶段、药品库是空的、onboarding 还重；我懂 IVF——一选就懂、自动排出清晰时间轴，还能拉伴侣进来。

v2 早孕 / 保胎（创始人 6 月底正用到） 大方向定 · 细节待讨论

↓ 以下功能为初步设想、尚未拍板，之后继续讨论。

核心功能（设想）：HCG 翻倍追踪 · 孕周 / 预产期 · 转产科节点提醒 · 保胎用药延续。

痛点 & 有多痛：验孕后盯着 HCG 数字焦虑、不知道翻倍正不正常、不知道何时该「转产科建档」。

用户现在怎么办：单独的 HCG 计算 App（做得全但和前面割裂）· 百度 · 群里问。

凭什么用我：和前面的用药 / 指标数据连贯——同一个病程管家一路走下来，不用换 App 重录，平滑接住「IVF→怀孕」这一段。

v3 孕期 大方向定 · 细节待讨论

（初步设想）产检节点提醒 · 孕期关键指标 · 用药安全。延续病程管家的「陪你走更久」。

横切 / 后续 随时加（均为方向、未细化）

多周期病程档案（v1 单周期之后扩展）· 夫妻共享完整版（导入式分享，UI 中性含同性伴侣）· 情绪陪伴重功能（冥想等）· 游戏化旅程大地图。

6 长期壁垒（护城河在哪）

「凭什么用我」已在第 4 节逐版本讲了。这里回答更难的问题：做起来后，凭什么别人抄不走。

诚实前提：单纯一个「确定性计算工具」技术门槛低、能被抄——单工具没有壁垒。护城河不在工具本身，在一套组合拳 + 时间。

壁垒是什么深度 / 时机
① 转换成本用户录的病程数据越积越多，迁移代价越大（数据结构按病程设计）中，随使用时长加深
② 内容 IP + 真实身份创始人是真实 IVF 患者、「为自己做的工具」——竞品能抄功能，抄不了这个信任强，独有，立刻生效
③ 社群口碑飞轮IVF「姐妹推荐」最高效，抢占「群里大家推的那个」心智位强，先发优势
④ 医生 / 诊所关系B2B2C；机构推荐 = 准入壁垒，竞品要撬动医生习惯最深，最难，需时间
⑤ 规模 + 可延伸性把切口做透 → 纵向延伸母婴 / 备孕 → 跟诊所绑定远期，方向性
⑥ 匿名聚合数据远期或成全国唯一真实患者数据库——但与「本地存储」有张力，先记着远期·存疑
最现实的护城河 = 工具（转换成本）+ IP（信任）+ 社群（口碑）的组合，三件事互相加强；医生关系是第四层。

但更本质的一条：与其焦虑「造一道强壁垒」，不如承认——愿意用心做 + 真的懂（你自己就是用户）+ 做出真实好用、体验好的产品，本身就是一种护城河。这正是苹果「体验即壁垒」的道理。先把产品做到真好用，壁垒往往是水到渠成的副产品；这也接住了「初心」——用心做一个自己都离不开的东西。

7 盈利方式（分阶段）

阶段做法目标
一（现在）免费上线。本地存储 + 确定性计算 + 豆包粘贴 ≈ 零成本。个人小程序暂收不了款（需营业执照 + 商户号）。验证价值、跑通端到端、不亏
二核心免费 + 便宜年费（如 59/年）解锁高级功能；补企业资质。收费本质是验证价值 + 自我造血
三B2B2C：医院 / 诊所把工具推荐给患者（冷启动 + 背书）。远期不排除被收购退出。规模化获客 / 退出想象空间
商业模式红线：不做医院引流（保中立），让医生敢推；不卖数据；首要目标是「不亏」。

8 用户隐私 & 合规

数据全存用户手机本地，不上服务器。

一句话卖点：「你的试管数据只存在你自己手机里。我们没有服务器、也看不到你的数据——技术上做不到，不是承诺不看。」这是架构层面的信任，不是一纸隐私政策。

为什么本地 = 最优解

  • 隐私（IVF 用户三重暴露风险：职场/家庭/社会，对隐私极度敏感）
  • 合规（微信小程序最简单）+ 低成本（第一版无后端）+ 中立
  • 代价（数据资产 / 变现）正是创始人本来就不要的

即使本地，仍不可缺（PIPL 健康数据全适用）

  • 隐私政策 / 首次单独同意 / 一键删除权
  • 微信「用户隐私保护指引」配置（硬门槛）
  • 本地加密；换机 / 清缓存会丢 → 必须做导出备份

完整法规依据见 隐私合规研究.md。

9 风险 & 红线

风险 / 红线应对
