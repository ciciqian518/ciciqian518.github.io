⛔ 三条贯穿红线(设计每一个功能时都要先过这三关)

  1. 只记录 + 提醒"医生定的方案",绝不自给医学/剂量建议 —— 一旦"推荐剂量/给用药建议"就可能落入二类医疗器械(NMPA)。守住"用户录入 → 我只排程+提醒+记录"。
  2. 确定性 > LLM —— 时间轴、提醒、记录全是确定性逻辑;每个关键数字都引导用户/医生 double check + 免责文案。
  3. 数据本地存储 —— 不上服务器,隐私即卖点;换机要靠导出备份(留接口)。

1 · 核心场景:一句话

把 IVF 患者手里"多种药 × 多种途径 × 多个时段 × 还隔天被医生改"的一堆乱药, 变成一条清晰的「今日时间轴」 + 该打/该吃时准时提醒 + 医生改方案时一键更新, 并能把这条时间轴分享给伴侣、让伴侣搭把手提醒。

北极星体验(Aha)

用户第一次打开就看到:
「今天 3 件事 —— 早 8:00 补佳乐 · 晚 8:00 黄体酮针 · 睡前 雪诺酮」
那种「一堆乱药 → 一眼看懂今天该干嘛」的解脱感 = 掌控感。这是整个产品最核心的情感承诺,demo 必须把这一下做出来。

2 · 按阶段拆:用药真实地图

用药贯穿 IVF 全程,但密度和痛点集中在三段:促排、夜针、移植后黄体支持。下表是"什么阶段、用什么药、管理到底乱在哪"。(药名/储存来自调研,剂量一律不预置,由医生方案录入——见红线①)

阶段典型药物(途径)用药管理乱在哪提醒的特殊性
降调/预处理达必佳等 GnRH-a(长效一针抵 18 天 / 短效每日皮下);保健品 CoQ10/DHEA/维D长效 vs 短效搞不清;复诊时间有要求(如上午 9 点前)不提前告知长效=单次大事件;短效=每日
促排(最痛)果纳芬/丽申宝/尿促(皮下注射) + 可能拮抗剂每天回家自打、"今天打几支/漏打怎么办";剂量医生当天口头改"加半支/减到一支",无文字记录易记错剂量隔天在变;多药并行
夜针/Trigger(最高危)HCG 夜针当天临时通知;误差需 ≤10 分钟,差几小时可能整周期取消精确到分钟 + 多重提醒 + 绝不可漏
取卵后视情况少量;OHSS 观察什么症状该去急诊边界不清用药少,提醒压力低
移植·内膜准备补佳乐/戊酸雌二醇/芬吗通(口服/阴道)每 3-5 天调剂量;内膜薄反复推迟移植日剂量动态变
移植后黄体支持
(密度最高)		黄体酮针(肌注,油剂不能冷藏) + 雪诺酮/安琪坦(阴道) + 地屈孕酮(口服)三路并行、持续 14 天→验孕→孕 12 周;副作用 vs 着床症状难分;"忘服孕酮怎么补"多药多途径多时段、长周期、不可漏
早孕/保胎 (v2)保胎用药延续 + HCG/孕周减药焦虑、何时转产科建档v2 扩展,留口子
储存提示要进药品库:果纳芬类需冷藏、黄体酮油剂不能冷藏——这是通用提醒 App 完全没有、而 IVF 真实需要的细节。药品库起步覆盖哪几种、要不要带储存提示?

3 · 按功能拆:三层(核心命门 / 轻量进 v1 / 留口子扩展)

🎯 核心命门 · v1 必做(差异化所在,国外也没做好)
F1 多药/多途径/多时段排程 → 自动生成「今日时间轴」:针/口服/阴道分类清晰,按早中晚摊开。这是 Aha 的载体。
F2 IVF 专用药品库:预置常见药的 名称/途径/储存提示(剂量不预置,由方案录入)。一选就懂,不用手填,这是"懂 IVF"的体现。
F3 方案随阶段/医生变 → 「一键更新」而非重建:旧方案作废、新方案生效、留变更记录。← 最大差异化命门,也最难,demo 必须体现这一下。
F4 准时提醒(含夜针精确提醒):到点响。
F4 技术硬骨头:安卓杀后台 / 提醒收不到,是国外 Bonzun 翻车的点。第一版(web/小程序)能做到什么可靠性?夜针这种高危的要不要降级方案(如多重冗余提醒 + 提前确认)?
F3 在 demo 里做到什么程度?真能改一套方案并刷新时间轴,还是先做"演示性"的一两步?
🪶 轻量进 v1(成本低、情感价值高)
F5 服药打卡(打了没/几点打):把"掌控感"变具体,也为依从性留数据。
F6 伴侣参与:把今日用药分享给伴侣 / 伴侣帮提醒——是情感价值的一部分,不是次要功能。
F7 漏服/错打的"怎么办"提示。
F6 v1 做到哪一步:只读分享给伴侣,还是双向(伴侣能反过来提醒你)?
F7 红线确认:漏服提示只能给"联系你的医生"这类中性引导,不能给"补服半支"之类医学补救建议——对吗?
🌱 留口子 · v1 后 / v2(留接口,不进 v1)
多周期病程档案 · 卵泡趋势 · HCG 追踪 · 报告解读 · 情绪陪伴重功能(冥想) · PGT 深分支 · 拍照识别医嘱 · 豆包粘贴导入 …

4 · 按体验拆

第一眼 = Aha:进 app 直接是"今日时间轴",不是空表格、不是先让你填一堆。
情感两支柱:① 自主掌控(被一堆药推着走 → 我掌控我的治疗) ② 伴侣参与(让伴侣看见这份不易、搭把手)。
语气与视觉:陪伴、不冰冷、不卖惨、不制造焦虑;温柔克制低饱和,避母婴卡通 + 避医疗冷。
onboarding 最短路径(医生诊室触点设想):扫码 → 5 分钟录入方案 → 看到清晰提醒。
v1 的录入方式选哪个起步:纯手动(从药品库选) / 豆包粘贴 / 拍照识别?——拍照 v1 可能太重,手动 + 药品库可能最稳。

5 · 数据骨架草图(直接喂给 vibe coding 的地基)

把数据结构先想对,是 vibe coding 不返工的关键。这是草图,留口子。

  • Cycle 周期 —— 含 stage 阶段(降调/促排/移植/黄体支持…),支撑"方案随阶段变"。
  • Regimen 用药方案 —— 版本化:医生一改 = 生成新版本、旧版存档。这是 F3「一键更新」和"多周期档案"能不能长出来的地基。
  • MedItem 药物条目 —— { 药名 · 途径(针/口服/阴道) · 剂量+单位 · 频次 · 时间点[] · 起止日 · 储存提示 · 阶段标签 }。
  • DoseLog 服药记录 —— { 关联药物 · 计划时间 · 实际时间 · 状态(已服/漏服/跳过) },支撑 F5。
  • Reminder 提醒 —— 由 MedItem + 时间点生成。
  • Partner 伴侣 —— 只读分享 / 提醒,支撑 F6。
核心设计抉择:方案变更用版本化(新版本+存档)还是直接覆盖改?我建议版本化——多花一点、但 F3 和多周期档案都靠它。明天确认。

6 · 2 周 demo 的"最小可跑通核心"

诚实说:2 周 + 边学 coding,不可能全做。建议 demo 锁定一条"最痛 + 最能体现差异化"的主线跑通端到端,其余留口子。先选一个锚点阶段:

候选锚点为什么选它代价/难点
A 促排期最痛节点;剂量天天改,最能体现 F3 差异化命门夜针"分钟级 + 整周期作废"的可靠性,对新手 2 周是硬骨头
B 移植后黄体支持用药密度最高、三路并行,最能体现 F1「乱药→时间轴」的 Aha;周期长、且与你当下早孕保胎连续,最容易自己 dogfood持续周期长,但提醒的"高危度"比夜针低,更适合新手起步
我的倾向(供参考,不替你定):demo 锚点选 B 黄体支持——最直接把 Aha(三路并行 → 清晰时间轴)做出来、最容易自己用上、可靠性压力比夜针小,更适合 2 周新手。把促排的"剂量动态调整"(F3)作为第二个要展示的差异化亮点。

Demo 最小闭环:从药品库选 3 种黄体支持药 → 录入一套方案 → 自动生成今日时间轴 → 准时提醒 → 打卡 → 医生改剂量时一键更新。这条跑通 = demo 成立。
