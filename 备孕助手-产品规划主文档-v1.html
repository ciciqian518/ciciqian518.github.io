备孕助手 · 产品规划主文档

v1 · 2026-06-18 · IVF 用药管理（移植后切入）· 求职作品集 to C 产品 · 本文件 = 产品规划唯一主文档
北极星顺序原则：一个场景（移植后）做到"自己离不开 + 病友愿意用"、拿到 PMF 信号，才谈延伸。 促排、孕期是 PMF 的奖励，不是预设的进度条。——这条贯穿下面所有取舍。

一 · MVP v1 范围（已锁定）

场景：只做「移植后黄体支持」单场景 —— 在其中同时演 ① 乱药→今日时间轴（Aha）② 方案随医生变·更新（用免疫科调药/停药演示）③ 复诊 overview。

周期：单周期跑通 + 数据结构按多周期/病程留接口。

为什么是移植后：用药密度最高（Aha 最强）· Cici 当下正用（真 dogfood）· 6/25 可向集爱医生展示 · 可靠性压力比促排夜针小，适合新手 coder。促排留作可嫁接的场景模块（见第五节）。

二 · MVP 功能清单（移植后单场景）

每条标注对应痛点（P1–P7，详见《移植后用药-痛点深访小结-v1》）。优先级 = Must / Should / Could / Won't-v1。
🎯 必做 Must —— demo 最小闭环，缺一不可
① 中文 IVF 移植后药品库：雪诺酮/黄体酮针/达芙通/芬吗通/环孢素… + 途径 + 储存提示；剂量不预置 〔差异化 F2 · 国内空白〕
② 录入一套方案：从药库选 + 填医嘱（剂量/时段/起止/停药条件）〔onboarding · P1/P2〕
③ 自动生成「今日时间轴」：晨/午/晚/睡前摊开，针/口服/阴道分清 〔F1 · Aha · P1〕
④ 准时提醒（基础 + 绑手机日历）〔刚需 table stakes〕
⑤ 打卡（已服 / 防重复服）〔F5 · P3〕
⑥ 停药节点自动处理：到期提醒/移除（"用药7天停""查HCG停"）〔F3-lite · P2〕
闭环一句话：从药库选药 → 录方案 → 今日时间轴 → 到点提醒 → 打卡 → 医生停药自动更新。这条跑通 = demo 成立。
💪 强烈想要 Should —— 差异化命门，时间够就进 demo
⑦ 方案更新 + 版本留痕：医生调药 → 一键更新时间轴、旧版存档 〔命门 F3 · P4/P5〕
⑧ 一键复诊 overview：当前在用一览 + 三方医嘱合并 〔P4/P5 · 对 6/25 集爱医生最加分〕
🌱 锦上添花 Could —— 留接口/展示性，不强求
⑨ 库存提醒（按频率算余量、提前≥3天）〔P7〕　⑩ 短信/钉钉式强提醒　⑪ 多周期历史一览（展示性）〔P4〕　⑫ 伴侣分享 / 仪式感回顾〔F6/P6〕
🚫 v1 不做 Won't —— 明确砍，归 v2+
促排全流程 / 夜针分钟级 · 预填确认（扫描医嘱单 OCR，体验优化层）· 跨医生自动整合（v1 靠手动录入合并）· 拍照识别 / 报告解读 / 概率预测 / 变现

三 · PMF 定义（什么程度算成功）

不是下载量。对这个小而垂直的产品，PMF 信号 = 下面三条出现任意一条。
1
你自己（真 dogfood）：移植后这段每天还在用、不回头翻小黑板。你就是最苛刻的第一用户。
2
病友：用完愿意继续用、或主动推荐给别的姐妹。
3
医生：愿意把它推荐给自己的病人（6/25 集爱是第一次试探）。

这呼应 Cici 定的"体验即壁垒"——先把产品做到自己离不开，壁垒是水到渠成的副产品。

四 · Roadmap 时间线

6 月压缩：MVP + quick demo 视频都在 6 月，早拿市场反应；延伸 GATE 在 PMF 之后。
① 6 月（即日–6/30）· 进行中
MVP v1 出炉
移植后 Must 六条 + 复诊 overview 跑通；自己 dogfood。本周（W2）= demo 功能 + UI 第一版（走 web，最快出 demo）。6/25 带版给集爱医生；月底 quick demo 视频发布（录屏走查即可）。
② 7 月初–中
种子用户测试（轻维护）
这段重点在基金助理，备孕助手不大改；给 3-5 个复杂情况病友试用；小红书发"手搓 IVF 用药管家"过程帖收反馈。
③ 8 月中–9 月初（W9–W12）
按反馈迭代核心
只迭代移植后核心场景（UI 升级 + 补缺 + eval），不扩场景；把②的反馈落进去。
④ 9 月初（W13）
正式市场测试
录正式 demo 视频上线（QQ糖 / IVF 小号）；看反馈、留存、有没有人主动来求用。
🚪 PMF Gate：有自然用户 / 转发 / 求用 = 拿到 PMF 信号
⑤ Q4 起（Phase 2，只做不学）
场景延伸（PMF 后才启动）
嫁接促排（早期漏斗入口）→ 之后孕期/产检（向后延伸）。过程篇内容、部署平台、AI 模型、收费验证，也都压到这里。
⚠️ 门槛：④拿到 PMF 才启动；没拿到就继续打磨核心，不堆场景

五 · 场景延伸地图（核心引擎 + 可嫁接模块）

促排不是"另一个产品"，是同一条旅程的更早入口。核心引擎阶段无关，每个场景 = 加一个药品库模块 + 该阶段特有功能。

← 向前延伸（更早获客）

降调 · 促排 · 取卵
促排 = 早期漏斗入口，更早抓住用户 + 陪走完整条线；还能抓到"攒胚/全胚冷冻/反复促排不立即移植"的子人群（移植后场景抓不到）。
促排特有：剂量动态调整 · 夜针（分钟级）。PMF 后嫁接。

● 核心锚点（MVP v1）

移植后黄体支持
用药密度最高、可 dogfood、demo 最强。先用它把引擎跑通、拿 PMF。

向后延伸（留存更久）→

早孕保胎 · 孕期 · 产检
陪同一个人走更久，破"用完即走"。Cici 自己的旅程＝产品路线图。
⚙️ 核心引擎（阶段无关，所有场景共用）：今日时间轴 · 提醒 · 打卡 · 方案版本化 · 病程档案。
→ 嫁接任何新场景 = 加「该阶段药品库模块 + 阶段特有功能」，不重写引擎。这就是"促排能单独成场景、再嫁接进来"的底座。

六 · 关键决策（今天共识）

提醒定位
刚需 / table stakes（必做且做好，可加短信/钉钉式强提醒），但不是差异化卖点。差异化在其之上的 6 件 IVF 专属事。
demo 平台
v1 走 web（最快出 demo + 录视频，不卡小程序/App Store 审核）。正式部署平台（小程序/App Store）压到 PMF 后。
AI 模型
v1 几乎全是确定性逻辑 + 本地存储，基本不调 LLM——"用哪个模型/要不要国外服务器"不是现在的必答题，留到 v2（扫描医嘱单/报告解读）再选型。
UI
功能先落地；拿德国 app mockup 当 UI 蓝本（风格已对：温柔克制低饱和）；首页"今日时间轴"那屏要好看（demo hero 镜头），其余够用即可，别手搓样式（用组件库/让 AI 生成）。
收费
方向对（收费=验证价值），但压到 PMF 后——过早收费会拦住早期用户、污染 PMF 信号。v1 demo 免费。¥20 终生 / 年费留作 PMF 后的价格探针。门槛：小程序个人收不了款；App Store iOS-only + $99 + 医疗审核更严。
数据
本地存储、无后端（隐私即卖点 + 零成本）；换机靠导出备份（留接口）。
合规红线
只记录+提醒"医生定的方案"，绝不自给医学/剂量建议（避二类医疗器械）；确定性 > LLM；所有数字引导跟医生 double check + 免责。

七 · 下一步：开写前先出「轻量 Build Spec」

这份规划文档够对齐和启动；但 vibe coding 前还需一份轻量 Build Spec（不是厚 PRD），补三样 AI 写代码最易跑偏的：

① 用户流程（一屏到一屏）　② 数据模型（周期/方案/药条/打卡/提醒——定死它，AI 才不反复重画）　③ 每个功能的验收标准　＋ 技术栈（web/本地/无后端/v1 不调 LLM）＋ mockup 引用。

协作方式（non-technical 友好）：开写前花一次 focus 把它落成 Build Spec → 然后一个功能一个功能、薄切片地建（先把"今日时间轴"用假数据渲染出来 → dogfood → 再加录入 → 再加提醒…），每切一片就自己用一次。
